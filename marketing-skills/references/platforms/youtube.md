# YouTube Playbook

How an agent optimizes YouTube videos for both YouTube search and AI-answer citation, working from the user's own channel data and public video pages. Title, thumbnail, and description form one packaging system; chapters and transcripts feed retrieval.

## Why YouTube matters for GEO

Google surfaces YouTube results in the main SERP, video mode, and Discover, so a ranking video earns two surfaces at once. YouTube also shows up heavily in AI answers: per the OtterlyAI YouTube Citation Study (2026), YouTube and Reddit together account for roughly 78% of social-media citations in AI Overviews, and YouTube citations rose ~25% since Jan 2025 — led by instructional and visual-demo content, with long-form videos dominating. Treat these as reported figures, not guarantees; AI surfaces shift fast.

## First, assess

| Input | Why it matters |
|-------|----------------|
| Video type | Tutorial / review / vlog / Shorts — sets length, hook, packaging |
| Primary + secondary keyword | One head term, a few supporting phrases |
| Audience | Existing subscribers vs. cold search/suggested traffic |
| Existing data | Pull CTR, avg view duration, traffic source from the user's own YouTube Studio (manual paste) |

## Packaging: title + thumbnail + description

These three reinforce one core message and target different parts of discovery. Build them together, not in isolation.

**Title**

| Rule | Guideline |
|------|-----------|
| Length | Aim under ~60 chars; mobile truncates beyond that |
| Keyword | Primary term near the front (first few words) |
| Frame | Lead with the payoff or curiosity gap, in plain language |
| Avoid | Keyword stuffing, ALL CAPS (except a single emphasized word) |

**Thumbnail**

| Rule | Guideline |
|------|-----------|
| Readability | High contrast, bold readable type, legible at thumbnail size |
| Composition | One clear subject, uncluttered; rule of thirds |
| Pairing | Don't repeat the title text — add a complementary idea |
| Iterate | Custom thumbnails are near-universal among top videos; test variants and watch CTR |

**Description**

| Rule | Guideline |
|------|-----------|
| First 2–3 lines | Primary + secondary keyword naturally; this is the visible snippet |
| Body | Plain-language summary so search and AI parsers grasp the topic |
| Links | Site, product, cited resources — relevant only |
| Chapters | Timestamped chapter list (see below) |

CTR and average view duration are the signals that compound — packaging earns the click, the video earns the watch time.

## Chapters & transcripts (retrieval surface)

Chapters and transcripts are the text that search engines and AI systems actually read. They make a video skimmable and quotable.

- **Chapters:** add timestamps in the description (`0:00 Intro`, `1:24 Step one`, …). First stamp must be `0:00`; minimum three, each ≥10s. Each chapter title should name the sub-topic in searchable words.
- **Transcript/captions:** upload an accurate caption file rather than relying on auto-captions. Clean text helps both accessibility and machine retrieval, and gives AI systems quotable passages.
- **Spoken hook:** state the topic and primary keyword out loud early — it lands in the transcript and matches voice/query intent.

## Tags

Tags are a minor signal; title, description, and on-screen/spoken content carry far more weight. Add a small set of relevant tags and stop. There is no proven ranking benefit to long tag lists.

## Length & hook

- **Hook:** earn the first ~15–30 seconds — name the payoff, no long intro. Early drop-off caps total watch time.
- **Long-form** tends to dominate AI citations for instructional topics (per the OtterlyAI study), so favor depth over padding for tutorial/explainer content; let Shorts handle discovery and reach.
- Match length to the format and to what the audience came for — there is no universal optimal duration.

## Output an agent produces

- 2–3 title options (under ~60 chars, keyword forward)
- Description: first 160-char snippet + full body with chapter timestamps
- Thumbnail brief: subject, on-image text, composition notes
- A small relevant tag set
- Caption/transcript reminder if none exists
