# Task 16: crew.py — Sequential Crew Orchestration

## What was created
`crew.py` — `InfluencerCampaignCrew` class with sequential Discovery → Proposal → Outreach pipeline.

## Class design
- `__init__`: lazy-load pattern — agents and crew are created inside `kickoff()`, not at init time.
- `_get_db()`: initializes `Database(AGENTS_DB_PATH)` once per instance, calls `init_db()`.
- `_track_tokens()`: accumulates `total_tokens` from CrewAI `token_usage`; warns via logger if `> MAX_TOTAL_TOKENS_PER_RUN`.
- `_safe_json_parse()`: strips markdown fences and parses JSON arrays from agent output.

## kickoff() flow
1. Insert brand_brief via `db.insert_brief()` → `brief_id`
2. Run Discovery task → ranked creators JSON array
3. Insert `campaign_suggestions` rows for each ranked creator
4. Run Proposal task on ranked creators → proposals JSON
5. Run Outreach task with `send` flag and `brief_id`:
   - If `approve_each=True`, prompts `input()` Y/N per creator before including in outreach
   - Parses outreach results for `dms_attempted` and `dms_sent` counts
6. Return summary dict with `brief_id`, `creators_found`, `suggestions_saved`, `dms_attempted`, `dms_sent`, `dry_run`, `total_tokens`

## Error handling
- Each step wrapped in try/except; exceptions are logged, not raised.
- Per-creator failures in suggestions insertion are caught individually.
- Crew never crashes on single-creator failure.

## Exclusions
- No Negotiator or Contract stage in this crew.
