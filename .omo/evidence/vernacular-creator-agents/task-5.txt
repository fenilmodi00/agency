## Task 5: Scraper DB/API Integration Tools

### File created
`tools/scraper_tools.py` — CrewAI `@tool` decorated functions for querying creator scraper data.

### Functions implemented
- `query_creators(filters: dict)` — supports SQLite (`SCRAPER_DB_PATH`) or HTTP API (`SCRAPER_API_URL`); default filters applied: follower_min=8000, follower_max=50000, min_reel_views=60000, is_active=true
- `get_creator_details(username)` — full creator record by username
- `get_creator_content_summary(username)` — content summary lookup
- `get_creator_language(username)` — language lookup
- `get_creator_recent_posts(username)` — recent posts lookup

### Design notes
- Conditional `crewai` import: falls back to a pass-through `@tool` decorator when crewai is unavailable, so the module imports and functions are callable regardless of install state.
- Read-only access: never modifies scraper data; all DB operations use SELECT-only queries.
- Fail-safe: returns `[]` or `{}` when neither backend is configured or when connections fail.
